# Rule Checklist

## 1. 사용 가능 여부

- [ ] Python 사용 가능 확인
- [ ] 외부 LLM API 사용 가능 여부 확인
- [ ] 외부 데이터 사용 가능 여부 확인
- [ ] 사전학습 모델 사용 가능 여부 확인
- [ ] 인터넷 검색 가능 여부 확인
- [ ] 제출 횟수 제한 확인
- [ ] 코드/PPT 제출 규칙 확인

## 2. Leakage 방지

- [ ] 평가 데이터 정답 추정 금지
- [ ] test/eval 데이터 수작업 라벨링 금지
- [ ] 리더보드 점수 기반 정답 분포 추정 금지
- [ ] 평가 데이터의 특정 패턴을 모델 구조/정답 후보 설정에 반영하지 않기
- [ ] train/validation/test 구분 명확히 기록

## 3. 재현성

- [ ] 최종 코드로 submission 생성 가능
- [ ] config 저장
- [ ] seed 저장
- [ ] 사용 모델/버전 기록
- [ ] 실험 로그 저장
- [ ] 실행 명령 기록

## 4. 본선 설명 가능성

- [ ] 왜 이 하네스 구조를 선택했는지 설명 가능
- [ ] 실패한 실험 기록 있음
- [ ] 최종 solver 선택 이유 있음
- [ ] 규칙상 위험을 어떻게 차단했는지 설명 가능

## 5. 자동 진행 금지 조건

다음 항목은 불명확하면 자동 진행하지 않는다.

- 외부 API 사용
- 외부 데이터 사용
- 인터넷 검색
- 평가 데이터 기반 패턴 분석
- 수작업 라벨링
- 리더보드 점수 기반 과최적화
